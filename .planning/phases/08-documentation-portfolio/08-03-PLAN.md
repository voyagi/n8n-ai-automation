**1. Title + One-line Pitch**
```markdown
# Contact Form AI Automation

**Automates contact form processing with AI-powered categorization, sentiment analysis, and smart routing — reducing per-submission handling from 15-28 minutes to under 10 seconds.**
```

**2. The Problem (2-3 sentences)**
Describe the manual pain: reading every form submission, categorizing manually, routing to the right person, logging in spreadsheets, drafting replies. At 50 submissions/day, that's 12-23 hours of repetitive work.

**3. The Solution (brief)**
What the automation does in one paragraph. Mention the key components: n8n workflow, OpenAI analysis, spam filtering, Google Sheets logging, Slack notifications.

**4. Business Impact Table**
Embed the key metrics table from docs/BEFORE-AFTER.md (the 6-row comparison table). Link to full ROI breakdown: `[See full ROI breakdown](docs/BEFORE-AFTER.md)`.

**5. Architecture**
Copy the Mermaid diagram from docs/ARCHITECTURE.md directly into the README (so it renders inline on GitHub). Add a link: `[Detailed architecture](docs/ARCHITECTURE.md)`.

**6. Features List**
Bullet list of capabilities:
- AI-powered categorization (support, sales, feedback, spam)
- Automatic spam filtering (>70% confidence threshold)
- One-line message summaries
- AI-drafted response suggestions
- Google Sheets logging with all submissions tracked
- Color-coded Slack notifications for legitimate inquiries
- Email notifications with HTML formatting
- Graceful error handling (always returns HTTP 200)
- Warnings tracking for partial failures (AI/Slack/Email)

**7. Quick Start**
Prerequisites list, then numbered steps:
1. Clone the repository
2. Install dependencies (`npm install`)
3. Copy `.env.example` to `.env` and fill in values
4. Start n8n (`npm run n8n`)
5. Import workflow (link to workflows/README.md for detailed steps)
6. Configure credentials (list the 4 types, link to workflows/README.md)
7. Start the form (`npm run dev`)
8. Test with sample data (`node tests/batch-submit.js`)

**8. Test Data**
Brief description of the 13-entry test dataset covering support, sales, feedback, spam, and edge cases (unicode, long messages, minimal input). Reference `tests/test-data.json` and `tests/batch-submit.js`.

**9. Tech Stack**
Simple table:
| Component | Technology |
|-----------|-----------|
| Automation | n8n (self-hosted) |
| AI | OpenAI GPT-4 |
| Form | HTML + Vanilla JS |
| Storage | Google Sheets |
| Notifications | Slack + Email |

**10. Project Structure**
```
public/              Contact form (HTML + CSS + JS)
workflows/           n8n workflow JSON + import guide
tests/               Test data + batch submission script
docs/                Architecture diagram + ROI analysis

```

**11. Customization**
Brief section showing the workflow is adaptable:
- Swap Slack for Microsoft Teams, Discord
- Swap Google Sheets for Airtable, Notion, Supabase
- Swap OpenAI for Claude, local LLM
- Adjust spam threshold, AI prompts, notification format

<task type="auto">
  <name>Task 1: Add Switch node and Spam Response node to workflow</name>
  <files>workflows/contact-form-ai.json</files>
  <action>
Insert a Switch node ("Route by Spam Score") between Parse AI Response and Success Response, and add a Spam Response node for the spam branch.

**Add "Route by Spam Score" node** to the nodes array:
- id: "route-spam"
- name: "Route by Spam Score"
- type: "n8n-nodes-base.switch"
- typeVersion: 3.2
- position: [1400, 300] (where Success Response currently is)
- Parameters:
  - mode: "rules"
  - rules.rules[0]: output 0, condition: `{{ $json.spam_score }}` > 70 (type: number, operation: gt)
  - options.fallbackOutput: "extra" (routes non-matching items to separate output)

**Add "Spam Response" node** to the nodes array:
- id: "spam-respond"
- name: "Spam Response"
- type: "n8n-nodes-base.respondToWebhook"
- typeVersion: 1.1
- position: [1630, 480] (below and right of Switch, parallel with Success Response)
- Parameters:
  - respondWith: "json"
  - responseBody: `={{ JSON.stringify({ spam: true, message: "Submission flagged as spam", category: $json.category, spam_score: $json.spam_score, spam_reason: $json.spam_reason }) }}`
  - options.responseCode: 200 (spam was processed successfully, not a client error)

**Move existing "Success Response" node** position to [1630, 300] (shift right to make room after Switch).

**Update connections:**
1. Change "Parse AI Response" connection: output 0 now connects to "Route by Spam Score" (instead of "Success Response")
2. Add "Route by Spam Score" connections:
   - main output 0 (spam, spam_score > 70) → "Spam Response" node
   - main output 1 (fallback, legitimate) → "Success Response" node
3. Remove old "Parse AI Response" → "Success Response" connection (replaced by Switch routing)

**Design decisions:**
- Use `>` (strictly greater than) for threshold: score of exactly 70 routes to legitimate branch (false negatives are safer than false positives for UX)
- Return 200 for spam (not 400): the form submission was valid and processed, AI determined it's spam
- Include spam metadata in response so frontend can differentiate spam from legitimate responses
- fallbackOutput: "extra" ensures items not matching any rule still get processed (routed to legitimate branch)

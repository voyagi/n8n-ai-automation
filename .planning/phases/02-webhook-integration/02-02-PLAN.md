<task type="auto">
  <name>Task 1: Refactor script.js with CONFIG, auth, timeout, and error handling</name>
  <files>public/script.js</files>
  <action>
Rewrite `public/script.js` with the following changes. Preserve ALL existing validation logic (blur tracking, field validation, has-blurred class management, send-another reset). Only modify the configuration and submission handling.

**1. Replace WEBHOOK_URL with CONFIG object at top of file:**

```javascript
const CONFIG = {
  webhookUrl: "http://localhost:5678/webhook/contact-form",
  webhookAuth: "REPLACE_ME_WITH_YOUR_TOKEN",
  timeout: 15000,
};
```

Comment above CONFIG explaining: webhook URL, auth token must match n8n Header Auth credential, and timeout in ms.

**2. Update form submit handler to use CONFIG and auth header:**

In the fetch call, add:
- `signal: AbortSignal.timeout(CONFIG.timeout)` for 15-second timeout
- `"X-Webhook-Auth": CONFIG.webhookAuth` header alongside Content-Type
- Use `CONFIG.webhookUrl` instead of `WEBHOOK_URL`

**3. Remove client-side timestamp from payload:**

Remove `timestamp: new Date().toISOString()` from the data object. The Set node in n8n adds `submittedAt` server-side (more reliable, trusted time).

**4. Parse and use webhook JSON response on success:**

After `response.ok` check:
```javascript
const result = await response.json();
```
Then populate the results card with data from `result`:
- Set category text: `result.category` (e.g., "Sales inquiry")
- Set response time text: `result.estimatedResponse` (e.g., "within 24 hours")

Use DOM elements by ID (see Task 2 for the HTML IDs):
- `document.getElementById("result-category")` → textContent = result.category or "Processing"
- `document.getElementById("result-response-time")` → textContent = result.estimatedResponse or "We'll be in touch soon"

**5. Differentiate error types in catch block:**

Replace the generic error message with specific messages:

```javascript
catch (err) {
  let errorMessage;

  if (err.name === "TimeoutError") {
    errorMessage = "The request took too long. The server may be busy — please try again.";
  } else if (err.message && err.message.includes("Failed to fetch")) {
    errorMessage = "Could not reach the server. Please check that n8n is running and try again.";
  } else {
    errorMessage = err.message || "Something went wrong. Please try again.";
  }

  errorBanner.textContent = errorMessage;
  errorBanner.classList.remove("hidden");
}
```

**6. Handle server validation errors (400 status):**

When `!response.ok`, try to parse the error JSON from the server:

```javascript
if (!response.ok) {
  const errorData = await response.json().catch(() => ({}));
  throw new Error(
    errorData.message || `Server responded with ${response.status}`
  );
}
```

This way, when the n8n IF node returns a 400 with `{ error: "Validation failed", message: "All fields are required..." }`, the user sees the server's message.

**7. Keep form data preservation behavior:**

DO NOT clear form on error (existing behavior already correct).
Only clear form and show success card on successful response.
Form fields should remain disabled during the fetch, re-enabled in finally block (existing behavior).

**Key constraint:** Preserve the existing validation system exactly (blur tracking via WeakMap, has-blurred class, field-error spans, validateField function). Only modify the CONFIG constant, fetch call, response handling, and error differentiation.

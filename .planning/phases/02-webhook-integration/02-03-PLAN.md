**Setup (one-time):**

1. Start n8n: `npm run n8n` (or ensure it's already running at http://localhost:5678)
2. In n8n UI: Workflows → Import from File → select `workflows/contact-form-ai.json`
3. In n8n UI: Create a Header Auth credential:
   - Go to Credentials → Add New → Header Auth
   - Name: `Contact Form Auth` (or any name)
   - Header Name: `X-Webhook-Auth`
   - Header Value: `REPLACE_ME_WITH_YOUR_TOKEN` (must match CONFIG.webhookAuth in script.js)
4. In the imported workflow, click the Webhook node → Authentication → select the credential you just created
5. Activate the workflow (toggle in top-right corner)
6. If CORS errors occur: stop n8n, set environment variable `N8N_ALLOWED_CORS_ORIGINS=http://localhost:3000`, restart n8n
7. Start the form server: `npm run dev` (serves at http://localhost:3000)

**Test 1: Valid submission**

1. Open http://localhost:3000
2. Fill in all fields (name, email, subject, message)
3. Click "Send Message"
4. Expected: Loading spinner appears, then success card shows with:
   - "Message received!" heading
   - Results card showing Category: "general" and Expected response: "within 24 hours"
5. Check n8n execution history: should show successful execution with all form data

**Test 2: Validation error (empty fields)**

1. Click "Send another message"
2. Using browser dev tools, disable the HTML `required` attribute on the name field
3. Leave name empty, fill other fields, submit
4. Expected: Error banner shows server validation error message ("All fields are required and email must be valid.")

**Test 3: Network error**

1. Stop n8n server (Ctrl+C)
2. Click "Send another message", fill in all fields, submit
3. Expected: Error banner shows "Could not reach the server. Please check that n8n is running and try again."
4. Verify form inputs are preserved (not cleared)

**Test 4: Send another**

1. Restart n8n and re-activate workflow
2. Submit a valid form
3. After success card appears, click "Send another message"
4. Expected: Form reappears empty, results card hidden

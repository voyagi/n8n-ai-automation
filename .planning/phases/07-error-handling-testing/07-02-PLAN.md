**2. Create `tests/batch-submit.js`**

Node.js script (CommonJS, no external dependencies â€” use native `fetch` available in Node 18+) with:

```javascript
// Usage:
//   node tests/batch-submit.js              # Sequential mode (1s delay)
//   node tests/batch-submit.js --rapid-fire # All at once (concurrent)

const testData = require('./test-data.json');

const WEBHOOK_URL = process.env.WEBHOOK_URL || 'http://localhost:5678/webhook/contact-form';
const WEBHOOK_AUTH = process.env.WEBHOOK_AUTH || 'demo-token-2026';
const DELAY_MS = 1000;
const RAPID_FIRE = process.argv.includes('--rapid-fire');
```

Features:
- `submitForm(data)` function: POSTs JSON with Content-Type and X-Webhook-Auth headers, measures duration, logs result with test case name and expected vs actual category, returns `{ success, duration, data, result }` object
- Sequential mode (default): Iterates test entries with `DELAY_MS` between each, uses `await` in for loop
- Rapid-fire mode (`--rapid-fire`): Fires all requests simultaneously with `Promise.all()`
- Summary at end: Total count, pass count, fail count, average duration
- Category accuracy check: Compare `result.category` with `data.expected_category` (informational only â€” AI may classify differently and that's OK for a demo)
- Handles errors gracefully: try/catch per request, continues even if individual submissions fail
- Uses `process.exitCode = 1` if any submission returned non-200 HTTP status (but NOT for category mismatches â€” those are informational)

No external dependencies (no node-fetch). Uses native `fetch()` from Node 18+.

<task type="auto">
  <name>Task 1: Rewrite JavaScript for validation, submission, and state management</name>
  <files>public/script.js</files>
  <action>
Rewrite `public/script.js` completely. The new script handles three concerns: field validation, form submission with loading state, and success/error state management.

**Constants and DOM references (top of file):**
```javascript
const WEBHOOK_URL = "http://localhost:5678/webhook/contact";

const form = document.getElementById("contact-form");
const submitBtn = document.getElementById("submit-btn");
const errorBanner = document.getElementById("error-banner");
const successCard = document.getElementById("success-card");
const sendAnother = document.getElementById("send-another");
const fields = form.querySelectorAll("input, textarea");
```

**1. Validation system — "reward early, punish late":**

For each field in `fields`:
- Track a `hasBlurred` flag per field (use a WeakMap or closure)
- On `blur` event: add `has-blurred` class to the field, then call `validateField(field)`
- On `input` event: only call `validateField(field)` IF field already has `has-blurred` class (reward early — clear errors as user types fix)

`validateField(field)` function:
- Get the error span: `form.querySelector(\`.field-error[data-for="${field.name}"]\`)`
- If `field.validity.valid`:
  - Set error span textContent to `""`
  - Remove `invalid` class, ensure `has-blurred` class present (for green border CSS)
- If NOT valid:
  - Determine specific error message based on `field.validity`:
    - `valueMissing`: Use field-specific messages:
      - name: "Please enter your name"
      - email: "Please enter your email address"
      - subject: "Please enter a subject"
      - message: "Please enter your message"
    - `typeMismatch` (email): "Please enter a valid email address (e.g., name@example.com)"
  - Set error span textContent to the message

**2. Form submission handler:**

On form `submit` event:
- `e.preventDefault()`
- First, validate ALL fields: iterate `fields`, add `has-blurred` to each, call `validateField`
- Check `form.checkValidity()` — if false:
  - Focus the first invalid field: `form.querySelector(":invalid")?.focus()`
  - Return early (don't submit)
- If valid, proceed with submission:

**Loading state (before fetch):**
- Set `submitBtn.disabled = true`
- Add `hidden` class to the button text span (or change textContent)
- Remove `hidden` class from `.spinner` inside button
- Show spinner: Plan 01 creates the button with `<span class="spinner hidden"></span><span class="btn-text">Send Message</span>`. Query both spans and toggle visibility:
    ```javascript
    const spinner = submitBtn.querySelector(".spinner");
    const btnText = submitBtn.querySelector(".btn-text");
    spinner.classList.remove("hidden");
    btnText.textContent = "Sending...";
    ```
- Disable all fields: `fields.forEach(f => f.disabled = true)`
- Hide any existing error banner: `errorBanner.classList.add("hidden")`

**Fetch call:**
```javascript
const data = {
  name: form.name.value.trim(),
  email: form.email.value.trim(),
  subject: form.subject.value.trim(),
  message: form.message.value.trim(),
  timestamp: new Date().toISOString(),
};

const response = await fetch(WEBHOOK_URL, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(data),
});

if (!response.ok) {
  throw new Error(`Server responded with ${response.status}`);
}
```

**On success:**
- Hide the form: `form.classList.add("hidden")`
- Hide error banner (if visible): `errorBanner.classList.add("hidden")`
- Show success card: `successCard.classList.remove("hidden")`
- Do NOT call `form.reset()` yet (user might want "send another" with similar data — but for this demo, reset is fine since success card replaces form)

**On error (catch block):**
- Show error banner above form:
  ```javascript
  errorBanner.textContent = `Failed to send: ${err.message}. Please check that n8n is running and try again.`;
  errorBanner.classList.remove("hidden");
  ```
- Form stays visible and filled (user can retry without re-typing)

**Finally block (always runs):**
- Re-enable all fields: `fields.forEach(f => f.disabled = false)`
- Reset button: hide spinner, show text, re-enable
  ```javascript
  spinner.classList.add("hidden");
  btnText.textContent = "Send Message";
  submitBtn.disabled = false;
  ```

**3. "Send another" handler:**

```javascript
sendAnother.addEventListener("click", (e) => {
  e.preventDefault();
  successCard.classList.add("hidden");
  form.classList.remove("hidden");
  form.reset();
  // Clear all validation states
  fields.forEach(field => {
    field.classList.remove("has-blurred");
    const errorSpan = form.querySelector(`.field-error[data-for="${field.name}"]`);
    if (errorSpan) errorSpan.textContent = "";
  });
});
```

**Important implementation notes:**
- Use `const` for all bindings, `let` only for the hasBlurred tracking if needed
- Use `async/await` for fetch, not `.then()` chains
- Plan 01 already creates the button with both `.spinner` and `.btn-text` spans — no HTML changes needed in this plan
- Error messages must be actionable: include what failed AND what to try ("Check that n8n is running")
- All field references use `field.name` attribute for error span lookup

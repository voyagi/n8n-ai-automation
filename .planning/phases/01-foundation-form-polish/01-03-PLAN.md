  <name>Task 1: Install dependencies and configure n8n environment</name>
  <files></files>
  <action>
1. Run `npm install` in the project root to ensure n8n and all dependencies are installed.

2. Check if `.env` already exists. If not, copy `.env.example` to `.env`:
   ```bash
   cp .env.example .env
   ```
   The .env.example already has sensible defaults (`N8N_BASIC_AUTH_USER=admin`, `N8N_BASIC_AUTH_PASSWORD=changeme`). These are fine for local development.

3. Verify `.env` is in `.gitignore` (it already is — just confirm).

4. Start n8n with `npm run n8n` (which runs `npx n8n start`). n8n will create its `.n8n/` data directory on first run (already in .gitignore).

5. Wait for n8n to finish starting (look for "n8n ready on" or "Editor is now accessible via" in output), then test that the dashboard is reachable:
   ```bash
   curl -s -o /dev/null -w "%{http_code}" http://localhost:5678/
   ```
   Expect a 200 or 302 (redirect to login). Any non-error HTTP status confirms n8n is running.

6. Stop n8n after verification (Ctrl+C or kill the process). The n8n server does not need to remain running — we just need to confirm it starts successfully.

**Do NOT commit .env to git.** Only .env.example is tracked.
